# External Blockers

This file exists to keep the program honest about what can and cannot be
completed by repository work alone.

## Legal And Licensing

1. confirm the release posture for OSM-derived datasets
2. finalize attribution text for public releases and API responses

## Deployment

1. choose and provision the public API runtime
2. establish production monitoring and error handling for public services

## Data Production

1. validate future releases against the documented contract
2. add stronger packaging beyond the CSV-first v1 artifact set
3. finalize long-term attribution wording once legal review is complete

## Community

1. enable the preferred public support surface
2. publish launch materials
3. collect external feedback from the first real users
