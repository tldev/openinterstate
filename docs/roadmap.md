# Roadmap

## Phase 0: Charter And Boundary

1. lock public naming
2. finalize the first public data contract draft
3. document licensing and attribution requirements
4. decide what remains application-specific

## Phase 1: Repo Bootstrap

1. baseline repository docs and templates
2. CI skeleton
3. issue labels and initial backlog
4. website repo bootstrap

## Phase 2: First Public Release

1. export the first named public datasets
2. ship a manifest and checksums
3. publish example consumption queries
4. validate release structure in CI

## Phase 3: Website MVP

1. launch a public home page
2. publish docs and downloads pages
3. expose the latest release clearly

## Phase 4: Public API

1. stabilize read-only lookup endpoints
2. map endpoints to release artifacts
3. publish API examples and rate-limit guidance

## Phase 5: Community Launch

1. publish launch materials
2. enable Discussions or equivalent public support surface
3. make the next release target visible

## Phase 6: Sustainability

1. review maintainer load
2. decide whether formal governance is needed
3. revisit legal entity setup only if operationally justified
