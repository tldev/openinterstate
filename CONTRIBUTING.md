# Contributing

OpenInterstate is still in early project-definition work.

For now, contributions are most useful in these areas:

1. naming and schema review
2. licensing and attribution review
3. release format feedback
4. documentation clarity
5. examples and developer ergonomics

## Before Opening A PR

1. check for an existing issue or discussion
2. keep changes scoped to a single concern
3. update docs when changing terminology or public contracts
4. call out licensing or attribution implications explicitly

## Pull Request Expectations

Every PR should explain:

1. what changed
2. why it changed
3. whether it affects the public data contract
4. whether it affects attribution or licensing obligations

## Early Stability Warning

Until the first public release is published, naming, schemas, and repository
layout should be treated as draft.
